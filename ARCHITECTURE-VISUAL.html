Key Design Patterns

Middleware-First RBAC

All routes protected at network boundary before reaching handlers. Double-checked at API level with requireAuth().

Soft Deletes

All major entities use deletedAt: DateTime? instead of hard deletes. Every query includes { deletedAt: null } filter.

Date-Based Sequential IDs

Human-readable IDs: PREFIX-YYYYMMDD-NNNN. 16 generators for different entities (MRN, APT, INV, etc.)

Singleton Prisma Client

Global caching prevents connection pool exhaustion during dev hot reload. Single instance in production.

Standardized API Responses

All endpoints return { success, data, error?, meta? }. Helper functions enforce consistency across 29+ routes.

Dual Zod Validation

Client-side (React Hook Form + Zod) + Server-side (Zod in API routes). Shared schemas for type safety.

Parallel Data Fetching

Promise.all() used in every page to fetch multiple resources simultaneously. Reduces page load time.

Component Composition

Consistent page structure: PageHeader + StatsCard + Tabs + Card + Table. Reusable across all 26 modules.
